01 사내 운영 앱
○ 추가 검토
목적
흩어진 운영 자료와 업무를 하나의 플랫폼에서 관리. 항상 최신버전 유지 + 이력 추적 가능한 구조.
포함 기능
신규입사자 온보딩 · 브랜드 가이드/업무 매뉴얼 · QSC 체크리스트 · 매출 보고/매장별 게시판
신규입사자 온보딩
입사자가 순서대로 따라갈 수 있는 온보딩 페이지.
브랜드 가이드 / 업무 매뉴얼
운영자가 자료를 등록하면 직원이 항상 최신 자료를 조회할 수 있는 구조.
QSC 체크리스트 + 부적합 알림
구글 폼 또는 웹폼으로 체크리스트 제출 → n8n이 부적합 항목 감지 시 슬랙 알림 자동 발송. 이력 관리 필요 시 DB 연동해서 추가 개발 필요.
데일리/주간 매출 보고/매장별 매출 아카이빙
개발은 가능 하나 사내 DB구축 필요. 단기간에는 협업툴로 관리하는 것을 권장
⚠ 계정 관리, DB 설계, 권한 구조 등 고려할 것이 많아 원하는 앱이 한 번에 나오기는 어려움.

💡 컨설팅 후 가장 쉽게 만들 수 있는 기능을 하나 정해서, 바이브코딩으로 가능한지 확인 후 1–2시간 핸즈온 세션으로 만들어보는 방식을 제안.

💬 커뮤니케이션은 슬랙 등 협업툴, 스케줄·근태는 Flex 등 외부 SaaS 활용 검토 — 직접 개발 범위에서 제외 권장.

03 온라인 가격 모니터링 자동화
자동화 실습 ○ 다나와 추가 검토
현재 문제
공구/라이브/행사 시 최저가 모니터링을 사이트별 수동 검색으로 확인 중 — 시간 소요 발생.
목표
기준가 이하 판매 발생 시 채널/판매가/링크 자동 알림.
네이버 쇼핑
API 연동 가능. n8n/Make로 워크플로우 구성 시도 가능. 10~30분 폴링 방식. 당일 완성보다 자동화 툴 경험 자체가 목적.
다나와
공식 API 없음 → Python 크롤러 필요. 서버 운영 + 유지보수 부담. 별도 검토 필요.
⚠ "실시간" 불가 — webhook 미지원. 10~30분 폴링이 현실적인 최선. 공구·라이브 기간에만 가동하면 무료 플랜으로 커버 가능.

06 채널별 매출/광고 대시보드
바이브코딩
현재 문제
채널별 매출·광고 데이터 엑셀로 관리. 한눈에 비교·분석하기 어려움.
참고
기존 엑셀/CSV 업로드 → 채널별 매출, 광고 성과 시각화. 데이터가 이미 있어 구조 단순.
채널별 매출, 광고 성과 등 기존 데이터가 잘 분류되어 있다면 단순 시각화 대시보드 구축은 바이브코딩으로 가능. 이전 데이터와 비교해야하는 아카이빙 등이 필요하면 DB 연동 등 추가개발 필요
